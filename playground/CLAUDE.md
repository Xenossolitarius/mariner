# Playground

Example microfrontend apps for testing and demonstrating Mariner.

## Apps

- `app1/` — Vue 3 SPA (JS config), imports `navigator:shared` for pinia store, lazy-imports `navigator:lazy`
- `app2/` — Vue 3 SPA with env vars (`.env`, `.env.test`), imports `navigator:shared` and `navigator:js-test`
- `app3/` — React 18 app with `createReactNavigator`
- `shared/` — Pinia store consumed by other apps via `navigator:shared`. Exports only `pinia` and `useCounter` (no wildcard re-export).
- `lazy/` — Lazy loading example (simple function export)
- `global-env/` — Environment variable handling example (mariner name: `envs`)
- `js-test/` — Plain JavaScript (non-TypeScript) example

## Testing Client

`client/` contains static HTML files that load all built microfrontends via importmap script tags. Used for local integration testing:

```bash
# From root
pnpm client:index    # Serve the test client
pnpm client:build    # Serve built output
```

## Every App Has

- `mariner.config.ts/js` — Mariner config via `defineMarinerConfig()`
- `navigator.ts/js` — Entry point exporting the navigator
- `package.json` — App dependencies (includes `mariner-fe: workspace:*`, `vite: ^8`)
- `vite.config.ts` — Standard Vite config (uses `rolldownOptions` for Vite 8)

## Cross-App Imports

Apps can import from each other using virtual modules:
```ts
import { pinia } from 'navigator:shared'
```

## Generated Files

`.mariner/` directories are generated by `mariner generate-types` — do not edit.

## Dependency Versions

- Vue 3.5+, Pinia 3.x, React 18
- `@vitejs/plugin-vue` 6.x, `@vitejs/plugin-react` 6.x
